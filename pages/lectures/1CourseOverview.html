Windows Setup: WSL ü™ü

Step 1: Install WSL and Linux

Installation Steps:

  1. Enable WSL: Open PowerShell as Administrator and run:
    wsl --install
  2. Restart your computer when prompted
  3. Install Ubuntu from the Microsoft Store
  4. Launch Ubuntu and create your Linux username/password
Why WSL?
  • ‚úÖ Linux-native environment matching production servers
  • ‚úÖ Access to Linux tools and utilities
  • ‚úÖ Near-native performance with WSL 2
  • ‚úÖ Seamless integration with Windows

Windows Setup: PostgreSQL üêò

Step 2: Install Database Server

In your WSL Terminal:

  1. Update package lists:
    sudo apt update
  2. Install PostgreSQL:
    sudo apt install postgresql postgresql-contrib
  3. Start the database service:
    sudo service postgresql start
  4. Verify installation:
    sudo -u postgres psql -c "SELECT version();"
Alternative Databases:
  • MySQL: sudo apt install mysql-server
  • MongoDB: Follow MongoDB's official Ubuntu installation guide

In-Class Exercise üèóÔ∏è

Build Your First Database 60 minutes

Exercise Overview

Today, you'll design and implement a small database for a library management system. This hands-on exercise will reinforce your environment setup and introduce core database concepts.

Learning Objectives

  • ‚úÖ Create and connect to a database
  • ‚úÖ Design and create tables
  • ‚úÖ Insert sample data
  • ‚úÖ Write basic SQL queries
  • ‚úÖ Understand relationships between tables

Part 1: Create Database 10 min

Step 1: Connect to PostgreSQL

sudo -u postgres psql

Step 2: Create the Library Database

CREATE DATABASE library_db;
\c library_db;

You should see: You are now connected to database "library_db"

Step 3: Create a Schema (Optional)

CREATE SCHEMA library;
SET search_path TO library;
üí° Tip: Use \dt to list tables and \d tablename to describe a table structure

Part 2: Create Tables 15 min

Create the Authors Table

CREATE TABLE authors ( author_id SERIAL PRIMARY KEY, first_name VARCHAR(50) NOT NULL, last_name VARCHAR(50) NOT NULL, birth_date DATE, nationality VARCHAR(50) );

Create the Books Table

CREATE TABLE books ( book_id SERIAL PRIMARY KEY, title VARCHAR(200) NOT NULL, isbn VARCHAR(13) UNIQUE, publication_year INTEGER, pages INTEGER CHECK (pages > 0), author_id INTEGER REFERENCES authors(author_id) );

Create the Members Table

CREATE TABLE members ( member_id SERIAL PRIMARY KEY, email VARCHAR(100) UNIQUE NOT NULL, first_name VARCHAR(50), last_name VARCHAR(50), join_date DATE DEFAULT CURRENT_DATE );

Insert Members

Part 4: Query Practice 15 min

Query 1: List all books with their authors

SELECT b.title, a.first_name || ' ' || a.last_name AS author FROM books b JOIN authors a ON b.author_id = a.author_id ORDER BY b.title;

Query 2: Find books published before 1900

SELECT title, publication_year FROM books WHERE publication_year < 1900 ORDER BY publication_year;

Query 3: Count books by each author

SELECT a.first_name || ' ' || a.last_name AS author, COUNT(b.book_id) AS book_count FROM authors a LEFT JOIN books b ON a.author_id = b.author_id GROUP BY a.author_id, a.first_name, a.last_name ORDER BY book_count DESC;

Part 5: Loans Table 5 min

Bonus: Create a Relationship Table

Create the Loans Table

CREATE TABLE loans ( loan_id SERIAL PRIMARY KEY, book_id INTEGER REFERENCES books(book_id), member_id INTEGER REFERENCES members(member_id), loan_date DATE DEFAULT CURRENT_DATE, return_date DATE, CONSTRAINT valid_dates CHECK (return_date > loan_date) );

Insert Sample Loans

INSERT INTO loans (book_id, member_id, loan_date, return_date) VALUES (1, 1, '2025-01-10', '2025-01-24'), (3, 2, '2025-01-15', NULL), (5, 1, '2025-01-20', NULL);

Query Active Loans

SELECT m.email, b.title, l.loan_date FROM loans l JOIN books b ON l.book_id = b.book_id JOIN members m ON l.member_id = m.member_id WHERE l.return_date IS NULL;
