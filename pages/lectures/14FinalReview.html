ROUND 1: Database Fundamentals

Lectures 1-2: File Systems vs. Databases

Question 1

Which of the following is NOT a major limitation of file-based systems?
A. Data separation and isolation
B. Improved data security
C. Data duplication and redundancy
D. Application-data dependence

Answer to Question 1

Correct Answer: B. Improved data security

Explanation: File-based systems actually have poor data security, not improved security. This is one of their major limitations!

The five major limitations of file-based systems are:

  • Data separation and isolation - Each program has its own files
  • Data duplication - Same data stored multiple times
  • Data dependence - Structure tied to application code
  • Incompatible file formats - Different programs, different formats
  • Fixed queries - Hard-coded data access patterns

Question 2

What are the two main components of a database system?
A. Tables and Queries
B. Database and DBMS
C. DDL and DML
D. Users and Administrators

Answer to Question 2

Correct Answer: B. Database and DBMS

Explanation: A database system consists of two key components:

  • Database: A shared collection of logically related data and its description
  • DBMS (Database Management System): Software that enables users to define, create, maintain, and control access to the database

Together, they provide controlled access, data independence, and efficient management of data!

Question 3

What does DDL stand for, and what is it used for?
A. Data Definition Language - defines database structure
B. Data Display Language - shows query results
C. Data Deletion Language - removes records
D. Database Design Logic - plans schemas

Answer to Question 3

Correct Answer: A. Data Definition Language

DDL (Data Definition Language) is used to define the database structure and schema. Key DDL commands include:

CREATE TABLE Movies (...)
ALTER TABLE Movies ADD COLUMN rating
DROP TABLE OldData

DML (Data Manipulation Language), on the other hand, is used to manipulate data: SELECT, INSERT, UPDATE, DELETE

ROUND 2: Architecture & Relational Model

Lectures 3-4: DBMS Structure & Relational Foundations

Question 4

What are the three levels of the ANSI-SPARC architecture?
A. User, Application, Database
B. External, Conceptual, Internal
C. Physical, Logical, Virtual
D. Client, Server, Storage

Answer to Question 4

Correct Answer: B. External, Conceptual, Internal

The ANSI-SPARC Three-Level Architecture:

  • External Level (View Level): Users' individual views of the database - different perspectives for different users
  • Conceptual Level (Logical Level): Community view - complete logical structure of the entire database
  • Internal Level (Physical Level): Physical representation - how data is actually stored on disk

Key benefit: Data independence! Changes at one level don't affect other levels.

Question 5

Which of the following is NOT one of the four controlled access features provided by a DBMS?
A. Security System
B. Integrity Constraints
C. Query Optimization
D. Concurrency Control

Answer to Question 5

Correct Answer: C. Query Optimization

The Four Controlled Access Features are:

  1. Security System: Protects the database from unauthorized access (authentication & authorization)
  2. Integrity Constraints: Ensures data accuracy and consistency (PRIMARY KEY, FOREIGN KEY, CHECK constraints)
  3. Concurrency Control: Manages simultaneous data access by multiple users (transactions & locks)
  4. Recovery Services: Restores the database after failures (backups & transaction logs)

Query optimization is important, but it's not one of the four controlled access features!

Question 6

In the relational model, what is a tuple?
A. A column in a table
B. A row in a table
C. A constraint on data
D. A relationship between tables

Answer to Question 6

Correct Answer: B. A row in a table

Key Relational Model Terminology:

  • Relation: A table with rows and columns
  • Tuple: A row in the relation (a single record)
  • Attribute: A column in the relation (a named property)
  • Domain: The set of allowable values for an attribute
  • Degree: The number of attributes (columns)
  • Cardinality: The number of tuples (rows)

Example: In a Movies table with 100 movies and 5 columns, the degree is 5 and the cardinality is 100!

Question 7

What type of key uniquely identifies each tuple in a relation?
A. Foreign Key
B. Candidate Key
C. Primary Key
D. Alternate Key

Answer to Question 7

Correct Answer: C. Primary Key

Types of Keys in the Relational Model:

  • Superkey: An attribute (or set) that uniquely identifies a tuple
  • Candidate Key: A minimal superkey (no unnecessary attributes)
  • Primary Key: The chosen candidate key for unique identification
  • Alternate Key: A candidate key that was not chosen as primary
  • Foreign Key: An attribute that references the primary key of another relation

Example: In a Members table, both email and member_id might be candidate keys, but we choose member_id as the primary key.

ROUND 3: Relational Algebra

Lecture 5: The Mathematical Foundation of SQL

Question 8

Which relational algebra operation selects specific rows from a relation based on a condition?
A. Projection ()
B. Selection ()
C. Union (*)
D. Join ()

Answer to Question 8

Correct Answer: B. Selection ()

The Five Basic Relational Algebra Operations:

  • Selection (): Chooses specific rows based on a condition (horizontal subset)
  • Projection (): Chooses specific columns (vertical subset)
  • Union (*): Combines tuples from two compatible relations
  • Set Difference (): Tuples in first relation but not in second
  • Cartesian Product (): All possible combinations of tuples from two relations

SQL equivalent: rating='PG'(Movies) is like SELECT * FROM Movies WHERE rating = 'PG'

Question 9

What type of join operation combines tuples from two relations based on a matching condition?
A. Natural Join
B. Theta Join
C. Equijoin
D. All of the above

Answer to Question 9

Correct Answer: D. All of the above

Join Operations in Relational Algebra:

  • Theta Join (): Combines tuples based on any comparison condition (=, <, >, etc.)
  • Equijoin: A special theta join where the condition uses only equality (=)
  • Natural Join: An equijoin that automatically matches on common attribute names and removes duplicate columns

Example: Movies Rentals naturally joins on movie_id if both tables have that attribute!

ROUND 4: Serverless APIs & SQL Fundamentals

Lectures 6-7: Building APIs & Writing Queries

Question 10

What is the correct order of clauses in a SELECT statement?
A. SELECT, WHERE, FROM, GROUP BY, HAVING, ORDER BY
B. SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY
C. FROM, SELECT, WHERE, GROUP BY, ORDER BY, HAVING
D. SELECT, FROM, GROUP BY, WHERE, HAVING, ORDER BY

Answer to Question 10

Correct Answer: B

The Correct SQL SELECT Order:

SELECT column_list
FROM table_name
WHERE row_condition
GROUP BY grouping_columns
HAVING group_condition
ORDER BY sort_columns

Memory trick: "Some Frogs Will Get Hungry Often"

Key difference: WHERE filters rows before grouping, HAVING filters groups after grouping!

Question 11

Which aggregate function would you use to find the total number of rentals in the Rentals table?
A. SUM(rental_id)
B. COUNT(*)
C. AVG(rental_id)
D. MAX(rental_id)

Answer to Question 11

Correct Answer: B. COUNT(*)

The Five Main Aggregate Functions:

  • COUNT(*): Counts all rows (including NULLs)
  • COUNT(column): Counts non-NULL values in a column
  • SUM(column): Adds up numeric values
  • AVG(column): Calculates the average of numeric values
  • MIN(column) / MAX(column): Finds minimum/maximum values
SELECT COUNT(*) AS total_rentals
FROM Rentals;

Question 12

What wildcard character is used with the LIKE operator to match any sequence of characters?
A. ? (question mark)
B. * (asterisk)
C. % (percent sign)
D. # (hash)

Answer to Question 12

Correct Answer: C. % (percent sign)

LIKE Pattern Matching Wildcards:

  • % (percent): Matches zero or more characters
  • _ (underscore): Matches exactly one character
-- Find movies starting with "The"
SELECT * FROM Movies WHERE title LIKE 'The%';

-- Find emails with exactly 3 characters before @
SELECT * FROM Members WHERE email LIKE '___@%';

ROUND 5: Advanced SQL Queries

Lecture 8: JOINs, Subqueries, and Set Operations

Question 13

What is the difference between INNER JOIN and LEFT OUTER JOIN?
A. No difference, they're the same
B. LEFT JOIN includes unmatched rows from the left table
C. INNER JOIN is faster
D. LEFT JOIN only works with two tables

Answer to Question 13

Correct Answer: B. LEFT JOIN includes unmatched rows

Types of Joins:

  • INNER JOIN: Returns only rows with matches in both tables
  • LEFT OUTER JOIN: Returns all rows from left table, plus matching rows from right (NULLs for non-matches)
  • RIGHT OUTER JOIN: Returns all rows from right table, plus matching rows from left
  • FULL OUTER JOIN: Returns all rows from both tables (NULLs where no match)
-- Shows all members, even those with no rentals
SELECT m.name, COUNT(r.rental_id)
FROM Members m
LEFT JOIN Rentals r ON m.member_id = r.member_id
GROUP BY m.name;

Question 14

What does the EXISTS operator do in a subquery?
A. Returns all rows from the subquery
B. Checks if the subquery returns any rows
C. Counts rows in the subquery
D. Joins tables together

Answer to Question 14

Correct Answer: B. Checks if the subquery returns any rows

EXISTS vs NOT EXISTS:

  • EXISTS: Returns TRUE if the subquery returns at least one row
  • NOT EXISTS: Returns TRUE if the subquery returns zero rows
-- Find movies that have been rented
SELECT title FROM Movies m
WHERE EXISTS (
  SELECT 1 FROM Rentals r
  WHERE r.movie_id = m.movie_id
);

Tip: EXISTS is more efficient than COUNT(*) > 0 because it stops as soon as it finds one row!

Question 15

Which set operation removes duplicate rows when combining results from two queries?
A. UNION ALL
B. UNION
C. INTERSECT
D. EXCEPT

Answer to Question 15

Correct Answer: B. UNION

Set Operations in SQL:

  • UNION: Combines results and removes duplicates
  • UNION ALL: Combines results and keeps duplicates (faster)
  • INTERSECT: Returns only rows that appear in both queries
  • EXCEPT (or MINUS): Returns rows in first query but not in second
-- Combine action and comedy movies (no duplicates)
SELECT title FROM Movies WHERE genre = 'Action'
UNION
SELECT title FROM Movies WHERE genre = 'Comedy';

ROUND 6: Performance & Data Definition

Lectures 9-10: Optimization & Database Structure

Question 16

What database object is used to speed up query performance by providing fast data lookup?
A. View
B. Index
C. Trigger
D. Stored Procedure

Answer to Question 16

Correct Answer: B. Index

Indexes: Data structures that improve query performance by providing fast lookups, similar to a book index!

-- Create an index on the email column
CREATE INDEX idx_member_email
ON Members(email);

Trade-off: Indexes speed up SELECT queries but slow down INSERT/UPDATE/DELETE operations because the index must be maintained!

Use EXPLAIN ANALYZE to measure query performance before and after adding indexes!

Question 17

What is a database VIEW?
A. A physical table stored on disk
B. A virtual table based on a stored query
C. A backup of the database
D. A user's permission level

Answer to Question 17

Correct Answer: B. A virtual table based on a stored query

Views: Virtual tables that dynamically show the result of a stored query. They don't store data themselves!

-- Create a view for available movies
CREATE VIEW AvailableMovies AS
SELECT movie_id, title, genre
FROM Movies
WHERE available = TRUE;

-- Query the view like a table
SELECT * FROM AvailableMovies;

Benefits: Simplify complex queries, provide security (hide sensitive columns), and maintain logical data independence!

Question 18

What happens when you try to delete a parent row that has child rows with a FOREIGN KEY constraint set to ON DELETE CASCADE?
A. The delete is rejected with an error
B. Only the parent row is deleted
C. Both parent and child rows are deleted
D. Child rows are set to NULL

Answer to Question 18

Correct Answer: C. Both parent and child rows are deleted

Referential Integrity Actions:

  • ON DELETE CASCADE: Automatically delete child rows when parent is deleted
  • ON DELETE SET NULL: Set foreign key to NULL in child rows
  • ON DELETE RESTRICT (default): Prevent deletion if child rows exist
  • ON UPDATE CASCADE: Update foreign key values when parent key changes
CREATE TABLE Rentals (
  rental_id SERIAL PRIMARY KEY,
  movie_id INTEGER REFERENCES Movies(movie_id)
    ON DELETE CASCADE
);

Question 19

What is a database TRIGGER?
A. A query optimization tool
B. Procedure that automatically executes in response to events
C. A type of constraint
D. A backup mechanism

Answer to Question 19

Correct Answer: B. Procedure that automatically executes

Triggers: Database procedures that automatically execute (fire) in response to specific events like INSERT, UPDATE, or DELETE.

-- Trigger to log movie updates
CREATE TRIGGER log_movie_changes
AFTER UPDATE ON Movies
FOR EACH ROW
EXECUTE FUNCTION log_change();

Common uses: Audit logging, enforcing complex business rules, maintaining derived data, and sending notifications.

Caution: Triggers add overhead and can make debugging harder - use them wisely!

Challenge Question 1

Write a query to find members who have NEVER rented a movie. Which approach is most efficient?
A. LEFT JOIN with NULL check
B. NOT EXISTS subquery
C. NOT IN subquery
D. Both A and B are equally good

Challenge Answer 1

Correct Answer: D. Both A and B are equally good

Method 1: LEFT JOIN with NULL check

SELECT m.name
FROM Members m
LEFT JOIN Rentals r ON m.member_id = r.member_id
WHERE r.rental_id IS NULL;

Method 2: NOT EXISTS subquery

SELECT name FROM Members m
WHERE NOT EXISTS (
  SELECT 1 FROM Rentals r
  WHERE r.member_id = m.member_id
);

Both are efficient! NOT IN should be avoided if the subquery column can have NULLs.

Challenge Question 2

In the ANSI-SPARC architecture, which type of data independence allows you to change the physical storage without affecting the logical schema?
A. Logical data independence
B. Physical data independence
C. External data independence
D. Conceptual data independence

Challenge Answer 2

Correct Answer: B. Physical data independence

Two Types of Data Independence:

  • Physical Data Independence: Changes to the internal/physical level (storage structures, file organization, indexes) don't affect the conceptual schema or application programs
  • Logical Data Independence: Changes to the conceptual/logical schema (adding attributes or tables) don't affect external views or application programs

Example: Adding an index (physical change) doesn't require changing any queries - that's physical data independence!

Goal: Logical independence is harder to achieve but both are crucial for maintainability!

Challenge Question 3

What's the difference between a stored procedure and a user-defined function?
A. Functions must return a value; procedures may not
B. Functions can be used in SELECT; procedures cannot
C. Procedures can modify database state; functions (typically) cannot
D. All of the above

Challenge Answer 3

Correct Answer: D. All of the above

Key Differences:

  • Functions MUST return a value (scalar or table); procedures may return nothing
  • Functions can be used in SELECT statements and WHERE clauses; procedures cannot
  • Functions should be deterministic and not modify database state (pure); procedures can have side effects
  • Functions are called as part of expressions; procedures are invoked with CALL or EXECUTE
-- Function: Returns a value, used in queries
SELECT calculate_rental_fee(movie_id) FROM Movies;

-- Procedure: Performs operations, called directly
CALL process_overdue_rentals();

Common Mistakes to Avoid

Watch Out For These:

  • Forgetting JOIN conditions: Always specify ON clause in JOINs
  • WHERE vs HAVING: WHERE filters rows, HAVING filters groups
  • COUNT(*) vs COUNT(column): COUNT(*) includes NULLs, COUNT(column) doesn't
  • UNION vs UNION ALL: UNION removes duplicates (slower), UNION ALL keeps them
  • NOT IN with NULLs: Can return unexpected results - prefer NOT EXISTS
  • Missing GROUP BY: If you use aggregate functions, non-aggregated columns need GROUP BY
